# PPT 配色设计原则

> 从 [Nord](https://github.com/arcticicestudio/nord)、[Catppuccin](https://github.com/catppuccin/catppuccin)、[Radix Colors](https://github.com/radix-ui/colors)、[Adobe Leonardo](https://github.com/adobe/leonardo) 等成熟设计系统提炼的 PPT 配色方法论。
> 
> 本文档是 `themed-cn-pptx` skill 的设计参考附录。

---

## 1. 色相约束 — 一个 deck 只走一个色相家族

**原则**：整个演示文稿围绕一个主色相（Hue）构建，所有颜色都从这个色相的变体出发。

**为什么**：
- Nord 的 16 色全部基于蓝/青冷色相（HSL Hue 210°-220°）
- Catppuccin 四个 flavor 虽然有 14 种强调色，但底色（Base/Mantle/Crust）都统一在蓝灰调（Hue 220°-240°）
- 人类视觉对色相一致性极为敏感——多色相混用 = 「乱」

**应用到 PPT**：
- 选定一个主色相角度 θ
- Dominant = θ 附近 ±15°
- Secondary = θ ± 30°~60°（邻近色）或 θ ± 150°~180°（互补色）
- 背景和文字色相偏移不超过 θ ± 20°

**色相策略选择**：

| 策略 | 视觉效果 | 适合场景 | 案例 |
|------|----------|----------|------|
| 单色相 + 明度变化 | 沉稳、统一 | 商务、学术、技术报告 | Nord |
| 邻近色（±30°） | 丰富但和谐 | 教育、产品介绍 | Radix Blue + Indigo |
| 互补色（±150°） | 活泼、对比强 | IP 主题、创意演示 | Miku 青蓝 + 粉红 |

## 2. 明度阶梯 — 同色相 3-5 级明度梯度

**原则**：每个色相需要 3-5 级明度（Lightness）变化，覆盖从深底到浅底的完整范围。

**参考模型 — Radix Colors 的 12 级明度体系**：

```
Step 1-2:  极深 → Dark BG / 深底封面
Step 3-4:  深   → 深色卡片、深色文字
Step 5-6:  中   → Muted text / 弱化文字
Step 7-8:  中浅 → Soft accent / 柔和点缀
Step 9-10: 浅   → Light BG / 内容页背景
Step 11-12:极浅 → 近白
```

**映射到 PPT 6 色槽**：

| 色槽 | 对应明度阶 | 典型 Lightness | 用途 |
|------|-----------|---------------|------|
| Dark BG | 1-2 | 10-20% | 封面/分隔/收尾深色页 |
| Muted text | 5-6 | 40-55% | 浅底上的正文文字 |
| Dominant | 4-5 | 50-60%（饱和色） | 品牌识别、条纹、装饰线 |
| Secondary | 4-5 | 50-60%（互补色相） | kicker、callout |
| Soft accent | 8-9 | 85-92% | chips、浅卡背景 |
| Light BG | 10-11 | 95-98% | 内容页底色 |

**关键约束**：
- 相邻明度阶之间至少差 **10% Lightness**，否则人眼分辨不出
- 但不要超过 **25%**，否则跳跃感太强
- Nord 的 Polar Night 序列：`#2e3440` → `#3b4252` → `#434c5e` → `#4c566a`，每级约增 8-10% Lightness

## 3. 饱和度策略 — 什么时候该灰

**原则**：大面积用低饱和度，小面积强调用高饱和度。

**Catppuccin 的启示**：
- 底色/背景色：Saturation 10-23%（极低，几乎灰）
- 文字/前景色：Saturation 10-16%（低，不抢眼）
- 强调色：Saturation 57-97%（高，立刻抓住注意力）

**应用到 PPT**：
- **背景色**：S ≤ 20%（近灰，不干扰阅读）
- **文字色**：S ≤ 25%（深灰/中灰，不刺眼）
- **品牌主色**：S 50-80%（饱和，但不荧光）
- **强调色/副色**：S 60-90%（最饱和，仅用于 kicker、callout）
- **绝对禁止**：S > 90% 的纯色用于大面积区域（PPT 投影后会更亮）

**饱和度陷阱**：
- 投影仪会放大高饱和度的刺眼感 → 现场演示时 Saturation 应比屏幕预览再降 10-15%
- 打印输出时高饱和度会偏色 → 如需打印，S 控制在 50% 以下

## 4. 对比度无障碍 — WCAG 标准

**原则**：文字色与背景色的对比度必须满足 WCAG AA 标准。

**WCAG 2.1 标准**：
| 级别 | 正常文字 | 大文字（≥18pt 或 14pt bold） |
|------|---------|--------------------------|
| AA（最低） | ≥ 4.5:1 | ≥ 3.0:1 |
| AAA（推荐） | ≥ 7.0:1 | ≥ 4.5:1 |

**对比度计算公式**（相对亮度法）：

```
contrast = (L_light + 0.05) / (L_dark + 0.05)

其中相对亮度 L 的计算：
1. 将 R/G/B 从 0-255 转为 0-1（除以 255）
2. 对每个通道：
   - 若 ≤ 0.04045: sRGB = value / 12.92
   - 若 > 0.04045: sRGB = ((value + 0.055) / 1.055) ^ 2.4
3. L = 0.2126 × R_sRGB + 0.7152 × G_sRGB + 0.0722 × B_sRGB
```

**PPT 中的关键检查点**：
- Muted text (如 `#59707B`) on Light BG (如 `#F1FBFA`) → 应 ≥ 4.5:1
- 白色文字 on Dark BG (如 `#0B1B2B`) → 应 ≥ 4.5:1
- Dominant 色块上的白色文字 → 应 ≥ 3.0:1
- **Secondary 绝不用于正文**，因为高饱和度互补色在浅底上通常不达标

**快速判断**：浅底 + 深字容易达标；深底 + 浅字需验证；**有色底 + 有色字最危险**。

## 5. 色彩心理学 — 场景→色温→色相

**原则**：不同演示场景对应不同的色温倾向。

| 场景 | 色温 | 推荐色相 | 代表色 | 情绪 |
|------|------|----------|--------|------|
| 科技 / SaaS / 基础设施 | 冷 | 蓝 210°-230° | `#5e81ac` | 信任、专业、冷静 |
| 金融 / 数据 / 分析 | 冷 | 深蓝 220°-240° | `#1e66f5` | 精确、可靠、权威 |
| 教育 / 培训 / 指导 | 暖偏中 | 绿 100°-160° | `#40a02b` → `#179299` | 成长、清新、开放 |
| 医疗 / 健康 / 生物 | 冷偏暖 | 青 170°-190° | `#88c0d0` | 清洁、安全、舒适 |
| 创意 / 设计 / 营销 | 暖 | 橙/粉 10°-40° | `#fab387` → `#f38ba8` | 活力、热情、创造 |
| 学术 / 论文 / 研究 | 中性 | 靛 230°-260° | `#8839ef` | 深度、思考、智慧 |
| IP 主题 / 二次元 | 自由 | IP 官方色 | `#39C5BB` | 以 IP 身份色为准 |

**色温混用禁忌**：
- 冷色主色 + 暖色副色 = 可以，但副色面积 ≤ 15%
- 冷色主色 + 冷色副色 = 安全，Nord 风格
- 暖色主色 + 冷色副色 = 可以，但需要足够对比度
- 多个暖色同时出现在不同层级 = 危险，容易显得「吵」

## 6. 配色生成算法 — 从一个种子色生成完整 6 色槽

**基于 Adobe Leonardo 的思路**：给定一个种子色（主色），自动推导其余 5 个色槽。

### 步骤

1. **提取种子色的 HSL**：
   ```
   seedHex → seedH, seedS, seedL
   ```

2. **Dominant 主色** = 种子色本身（或微调 Saturation 到 50-80%）

3. **Secondary 副色**：
   - 互补策略：`H = seedH + 150°~180°`（mod 360），S 不变或略降，L 不变
   - 邻近策略：`H = seedH + 30°~45°`，S 不变，L 不变

4. **Dark BG 深底**：
   - `H = seedH ± 10°`（微偏移），`S = 15-25%`，`L = 10-18%`

5. **Light BG 浅底**：
   - `H = seedH ± 5°`，`S = 10-20%`，`L = 95-98%`

6. **Soft accent 柔和点缀**：
   - `H = seedH`，`S = 15-25%`，`L = 88-93%`

7. **Muted text 弱化文字**：
   - `H = seedH ± 15°`，`S = 15-25%`，`L = 40-55%`

### 验证

生成后必须检查：
- Dark BG 上的白字对比度 ≥ 4.5:1
- Light BG 上的 Muted text 对比度 ≥ 4.5:1
- Dominant 色块上的白字对比度 ≥ 3.0:1

## 7. 常见配色失误

| 失误 | 表现 | 修正 |
|------|------|------|
| **彩虹 PPT** | 每页一个颜色，无统一主色 | 锁定一个色相，用明度/饱和度变化 |
| **纯黑纯白** | `#000000` 背景 + `#FFFFFF` 文字 | 深底用带色相的深色如 `#0B1B2B`，浅底用 `#F1FBFA` |
| **荧光色大面积** | `#00FF00`、`#FF00FF` 作背景 | 高饱和度仅用于 ≤10% 面积的强调元素 |
| **灰底灰字** | 中灰背景 + 深灰文字 | 确保 Lightness 差 ≥ 30%，或用对比度公式验证 |
| **副色用于正文** | 粉色/橙色大段文字 | 副色仅用于 kicker/callout，正文用深色或 Muted |
| **投影仪未考虑** | 笔记本上完美，投影后看不清 | 所有颜色 Saturation 降 10-15%，对比度用 AA 级别 |
| **忘加半透明遮罩** | 背景图上直接放文字 | 深色遮罩 opacity 40-50% 确保可读 |
